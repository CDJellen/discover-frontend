<h1><a id="user-content-frida" class="anchor" aria-hidden="true" href="#frida"><span aria-hidden="true" class="octicon octicon-link"></span></a>Frida</h1>
<p>Dynamic instrumentation toolkit for developers, reverse-engineers, and security
researchers. Learn more at <a href="https://frida.re/" rel="nofollow">frida.re</a>.</p>
<h1><a id="user-content-two-ways-to-install" class="anchor" aria-hidden="true" href="#two-ways-to-install"><span aria-hidden="true" class="octicon octicon-link"></span></a>Two ways to install</h1>
<h2><a id="user-content-1-install-from-prebuilt-binaries" class="anchor" aria-hidden="true" href="#1-install-from-prebuilt-binaries"><span aria-hidden="true" class="octicon octicon-link"></span></a>1. Install from prebuilt binaries</h2>
<p>This is the recommended way to get started. All you need to do is:</p>
<pre><code>pip install frida-tools # CLI tools
pip install frida       # Python bindings
npm install frida       # Node.js bindings
</code></pre>
<p>You may also download pre-built binaries for various operating systems from
Frida's <a href="https://github.com/frida/frida/releases">releases</a> page on GitHub.</p>
<h2><a id="user-content-2-build-your-own-binaries" class="anchor" aria-hidden="true" href="#2-build-your-own-binaries"><span aria-hidden="true" class="octicon octicon-link"></span></a>2. Build your own binaries</h2>
<h3><a id="user-content-dependencies" class="anchor" aria-hidden="true" href="#dependencies"><span aria-hidden="true" class="octicon octicon-link"></span></a>Dependencies</h3>
<p>For running the Frida CLI tools, e.g. <code>frida</code>, <code>frida-ls-devices</code>, <code>frida-ps</code>,
<code>frida-kill</code>, <code>frida-trace</code>, <code>frida-discover</code>, etc., you need Python plus a
few packages:</p>
<pre><code>pip install colorama prompt-toolkit pygments
</code></pre>
<h3><a id="user-content-linux" class="anchor" aria-hidden="true" href="#linux"><span aria-hidden="true" class="octicon octicon-link"></span></a>Linux</h3>
<pre><code>make
</code></pre>
<h3><a id="user-content-apple-oses" class="anchor" aria-hidden="true" href="#apple-oses"><span aria-hidden="true" class="octicon octicon-link"></span></a>Apple OSes</h3>
<p>First make a trusted code-signing certificate. You can use the guide at
<a href="https://sourceware.org/gdb/wiki/PermissionsDarwin" rel="nofollow">https://sourceware.org/gdb/wiki/PermissionsDarwin</a> in the sections
“Create a certificate in the System Keychain” and “Trust the certificate
for code signing”. You can use the name <code>frida-cert</code> instead of <code>gdb-cert</code>
if you'd like.</p>
<p>Next export the name of the created certificate to relevant environment
variables, and run <code>make</code>:</p>
<pre><code>export MACOS_CERTID=frida-cert
export IOS_CERTID=frida-cert
export WATCHOS_CERTID=frida-cert
export TVOS_CERTID=frida-cert
make
</code></pre>
<p>To ensure that macOS accepts the newly created certificate, restart the
<code>taskgated</code> daemon:</p>
<pre><code>sudo killall taskgated
</code></pre>
